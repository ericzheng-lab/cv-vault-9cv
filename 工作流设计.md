# CV 工作流设计文档

> 版本: v1.0 | 日期: 2026-04-06 | 类型: 人机协作

---

## 核心原则

**不是纯自动化，而是 AI 辅助 + 人工决策**

```
JD → AI 分析 → 人工确认 → AI 生成 → 人工微调 → PDF
         ↑                    ↑
      提供建议              初稿
      解释原因              优化建议
```

---

## 工作流步骤

### Step 1: 岗位分析 (`analyze-job.py`)

**输入**: JD (Markdown 文件)  
**输出**: `analysis.md` (分析报告)

**AI 做什么**:
- 提取 JD 关键词
- 判断岗位类型 (film/streaming/gaming/commercial/operations/strategy)
- 从数据库匹配最相关的经历和项目
- 生成策略建议 (强调/弱化/补充)

**人工做什么**:
- 阅读分析报告
- 确认或调整策略
- 添加额外要求
- 标记最终确认

**输出示例**:
```markdown
## 岗位类型判断
**主要类型**: Streaming Content
**置信度**: 8/10

## 策略建议
### 强调
- [ ] 原创 IP 开发
- [ ] 系列内容制作

## 推荐经历 (Top 5)
1. First Light Films - Led Global Co-Production
   - 匹配度: 9/10
   - 匹配标签: film, international, financing
```

---

### Step 2: 简历生成 (`generate-cv.py`)

**输入**: `analysis.md` (已确认)  
**输出**: `{company-position}/resume.html`

**AI 做什么**:
- 读取 `_resume-template.html`
- 根据确认的策略选择 bullets
- 重写 Summary
- 调整 Competencies 排序
- 填充模板

**人工做什么**:
- 查看生成的 HTML
- 修改语气和重点
- 添加特殊要求
- 确认终稿

---

### Step 3: PDF 导出 (`export-pdf.py`)

**输入**: `{company-position}/resume.html`  
**输出**: `{company-position}/resume.pdf`

**自动化**:
- Playwright 打开 HTML
- 导出 PDF
- 控制单页/多页

---

### Step 4: 投递记录

**自动更新** `submissions.json`:
```json
{
  "id": "2026-04-06-netflix-producer",
  "date": "2026-04-06",

```

## 脚本功能

### `analyze-job.py`

```bash
# 分析 JD，输出到控制台
python3 scripts/analyze-job.py --job 03-生成中/netflix/jd.md

# 分析并保存报告
python3 scripts/analyze-job.py --job jd.md --output analysis.md

# 输出 JSON 格式（供其他脚本使用）
python3 scripts/analyze-job.py --job jd.md --json
```

**功能**:
- 提取 JD 关键词（film, streaming, gaming 等 10 类）
- 判断岗位类型及置信度
- 从数据库匹配 Top 5 经历和 Top 3 项目
- 生成策略建议（强调/弱化/补充）
- 提供执行清单

## 人机协作点

| 步骤 | AI 角色 | 人工角色 |
|------|---------|---------|
| **岗位分析** | 提取关键词、匹配经历、建议策略 | 确认策略、调整方向 |
| **内容选择** | 按标签推荐 bullets | 添加/删除、调整顺序 |
| **Summary 重写** | 基于 JD 生成初稿 | 修改语气、压缩/扩展 |
| **生成后优化** | 提供改进建议 | 最终定稿 |
| **特殊需求** | 执行标准流程 | 处理非标准情况 |

---

## 保留的灵活性

1. **覆盖 AI 选择** — 可以强制使用/不使用某个 bullet
2. **添加新内容** — 数据库里没有的临时内容
3. **调整语气** — 正式/亲和/激进等不同风格
4. **特殊格式** — 某些岗位需要的特殊排版
5. **多版本对比** — 生成 A/B 版本供选择
